# Answer 1

## Open Traceability Assessment — strong OpenSustain.tech cases

I treated this as a **desk assessment**, not a full reproducibility audit. The HackMD definition frames Open Traceability as the degree to which a sustainability claim’s evidence chain is externally inspectable, across open input data, software/model, execution, review/community, publications, and linkage quality. It also says linkage is decisive: openness alone is not enough if the claim is not tied to exact datasets, code versions, workflow runs, review records, and publications. ([HackMD][1])

**Scale:** 0–5 per dimension.
**Final Total Linkage Score:** `(Open Data + Open Model + Open Execution + Open Community + Open Publication + Linkage Quality) / 30 × 100`.

### Main evidence basis

OpenSustain.tech lists the assessed energy-system modelling projects including PyPSA-Eur, PyPSA-Earth, OSeMOSYS Global, Euro-Calliope, GenX, and Open Energy Platform; it also lists Electricity Maps and Open Grid Emissions under emissions/carbon accounting, OpenAQ under air quality, Open Infrastructure Map under grid/infrastructure, and Brightway2 under life-cycle assessment. ([opensustain.tech][2])

**Top tier:** Open Grid Emissions, PyPSA-Eur, and PyPSA-Earth are the strongest cases because they connect open code, open or well-documented input data, executable workflows, publications/docs, and active issue/community processes. Open Grid Emissions is especially strong because its repository explicitly links open-source processing tools, public datasets, Zenodo archives of versions/intermediate outputs, methodology documentation, and a runnable pipeline. ([GitHub][3]) PyPSA-Eur is similarly strong: it describes an open model dataset built from open data using Snakemake, identifies major input sources, links to papers, issues, Discord, and licensing caveats for data sources. ([GitHub][4]) PyPSA-Earth links open data retrieval, model scope, publications, open meetings, public materials, and runnable Snakemake test commands. ([GitHub][5])

**Strong but less claim-specific:** OSeMOSYS Global, GenX, Euro-Calliope, and Open Energy Platform score well, but their traceability often depends on the specific study or scenario built with the tool. OSeMOSYS Global is explicitly open-source/open-data and links documentation, model structure, licensing, and a Nature publication, but exact generated runs still depend on user configuration. ([GitHub][6]) GenX has excellent model openness, documentation, example systems, and publication links, but user-supplied data and scenario configurations determine the claim chain. ([GitHub][7]) Euro-Calliope provides workflow routines, pre-built model DOIs, documentation, and a scientific article, though the latest release visible in the repository is older. ([GitHub][8]) Open Energy Platform is designed around quality, transparency, reproducibility, community data, metadata, API access, licensing, and citation, which gives it high linkage potential as research infrastructure. ([GitHub][9])

**Useful but with notable linkage gaps:** OpenAQ, Electricity Maps, Brightway2, and Open Infrastructure Map are valuable open sustainability infrastructure, but they are less complete as claim-level traceability chains. OpenAQ has open API access, source code, local development instructions, platform components, and licensing metadata for API data, but provider-level licences and measurement provenance vary by source. ([GitHub][10]) Electricity Maps has open parsers, source documentation, issues, discussions, and a transparent carbon-intensity rationale, but its platform includes non-open components and access to historical data/API is not equivalent to a fully open reproducible claim chain. ([GitHub][11]) Brightway2 is excellent open LCA software, but the traceability of an actual LCA claim depends heavily on the inventory database, impact method, foreground model, and run configuration used. ([GitHub][12]) Open Infrastructure Map is open-source and based on OpenStreetMap infrastructure data, but the exact extract/version and claim-level publication chain are usually weaker than in formal modelling workflows. ([GitHub][13])

## Interpretation

The strongest Open Traceability cases are not merely “open source”; they are **open source plus evidence-chain infrastructure**: documented inputs, executable workflows, archived releases, visible review, and publications that point back to the artifacts. By that standard, **Open Grid Emissions, PyPSA-Eur, and PyPSA-Earth** are the best candidates for full Claim Openness and Traceability Profiles.

## Total score

**Overall Open Traceability score: 84 / 100**

PyPSA-Eur is a highly traceable open energy-system modelling project. Its strongest areas are **open source software, licensing, workflow transparency, data-source documentation, and public review infrastructure**. The main gap is **claim-level linkage**: for a specific environmental claim, the ideal traceability package would include the exact release/tag, data-source versions, configuration file, scenario file, solver/environment details, workflow logs, outputs, publication, and review/discussion links in one externally inspectable evidence chain.
